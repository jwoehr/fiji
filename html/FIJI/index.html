What is FIJI?

[Go Back to Table of Contents]

FIJI is a forth-like interpretive procedural language which is in reality a Java interpreter. You can load and exercise Java classes interactively in FIJI, or you can compile FIJI programs to do so.

FIJI is an open source project licensed under the GNU Public License.

Where do I get FIJI?

[Go Back to Table of Contents]

FIJI is now a SourceForge project.

The latest FIJI release is also linked from the SoftWoehr Home Page.

How is FIJI distributed?

[Go Back to Table of Contents]

FIJI is distributed in a .zip file fiji.version.zip and unpacks to the following structure:

+- com (directory)
|
+- html (directory)
|
+- scripts (directory)
|
+- fiji.version.jar

Where is the Source Code?

[Go Back to Table of Contents]

This one question has been asked over and over again!

The source code is packed into the .jar file with classes!

Please use the Java jar tool, e.g.

jar xf fiji.version.jar

to unpack the source code if you wish to browse or recompile SoftWoehr Java class libraries.

How Do I Build FIJI?

Notice that in the same directory with the FIJI jar file, once you have unpacked the source from the jar, there is a directory called scripts in which is located a Jakarta Ant build file build.xml which is used to build FIJI. Presuming you have unpacked the FIJI source from the jar file as described above, then in that same directory invoke ant as follows:

ant -f ./scripts/build.xml

This will build the default target all which includes generating the FIJI jar file. Try:

ant -f ./scripts/build.xml usage

to see a usage message on all the targets you may invoke when using the supplied build file.

Where are the Examples?

[Go Back to Table of Contents]

The examples are in the same place the source code is: in the .jar file with classes.

Please use the Java jar tool, e.g.

jar xf fiji.jar

to unpack the examples found under com/SoftWoehr/FIJI/examples.
